# Linux 原生环境强杀特定进程

## 1 查询含有关键词的进程并强杀

通过关键词定位进程 ID (PID) 并将其传递给 `kill` 命令。

* **查询进程：**

```bash
ps -ef | grep "关键词" | grep -v grep
```

* **一行命令强杀：**

```bash
    ps -ef | grep "关键词" | grep -v grep | awk '{print $2}' | xargs kill -9
```

> **解释：** `ps -ef` 列出所有进程，`grep -v grep` 是为了排除掉 grep 命令本身的进程，`awk` 提取第二列的 PID，最后通过 `xargs` 传给 `kill -9`。

## 2 查询占用特定端口的进程并强杀

如果你已知端口号（例如 `8080`），可以使用 `lsof` 或 `fuser`。

* **方法 A：使用 `lsof` (推荐)**
    * **查询进程：**

        ```bash
        lsof -i :8080
        ```

    * **一行命令强杀：**

        ```bash
        lsof -ti :8080 | xargs kill -9
        ```

        > **注：** `-t` 参数会让 `lsof` 仅输出 PID，非常适合配合 `xargs` 使用。

* **方法 B：使用 `fuser`**
    * **查询并强杀：**

        ```bash
        fuser -k -9 8080/tcp
        ```

        > **解释：** `-k` 代表 kill，`-9` 代表强制信号。

## 3 更现代/简洁的工具：`pkill`

如果你只需按名称杀掉进程，`pkill` 是最快的选择。

* **强杀包含关键词的所有进程：**

    ```bash
    pkill -9 -f "关键词"
    ```

    > **警告：** 使用 `-f` 会匹配完整的命令行，请确保关键词足够精确，以免误杀其他系统进程。

## 4 💡 提示

* **权限：** 如果进程属于 root 或其他用户，你需要在命令前加 `sudo`。
* **确认：** 在执行“强杀”命令前，建议先执行“查询”命令，确认 PID 是否为你想要终止的那个，避免引发系统不稳定。
